## 3. 공통 런치 시퀀스 (재사용 템플릿)

출시는 *버스트가 아니라 웨이브*. 0→10K 스타 리포의 87%가 HN 먼저 → Reddit·X 교차
( `03` ). 자산 종류 무관 재사용:

1. **웨이브 0 (T-7~3일)** — 완성도 점검(라이선스·작동 코드·데모·7초 README) +
   네트워크에서 초기 신호(스타·좋아요) 확보. 소형 채널 예열.
2. **웨이브 1 (T-0, 화~목)** — 메인 채널 1 + 보조 1만. 동시 살포 금지.
   도구·포맷·리포→Show HN / 모델→r/LocalLLaMA·HF Post / 한국→GeekNews Show GN.
3. **웨이브 2 (T+1~7일)** — 댓글·이슈·Discussion **24시간 내 응대**. awesome-list PR.
4. **반감기 리셋 (T+6주)** — 후속 릴리스로 관여 반감기(약 6주, `01`)를 리셋.

## 4. 측정 지표

- **1차** — 자산별 다운로드/스타, 좋아요, 파생물(derivative) 수, Discussion 활동.
- **2차** — 외부 유입: HN 포인트, r/* 업보트, Space 방문, Ollama pull, awesome-list 유입.
- **선행** — 양자화 종수·채널 노출 수·카드 완성도·이슈 응답 시간.
- **리뷰** — 주 1회 채널별 기여 점검 → 배분 조정. 6주마다 케이던스 점검.

## 5. 안티패턴 — 바이럴을 죽이는 것

- **H1. 라이선스 누락** — Gemma 등 베이스 라이선스·use policy 동봉 필수. 누락=최단 takedown.
- **H2. 가짜 다운로드/봇/업보트 매수·투표 서클** — 플랫폼 탐지 → 섀도밴·도메인 영구밴.
- **H3. 유해 예시 출력 호스팅** — 데모는 엣지하되 방어 가능선. 안전 섹션 정직히.
- **H4. 벤치 과장** — 스크린샷으로 검증당함. 재현 가능한 수치(KL·거부율·MMLU)만.
- **H5. 한 방 노이즈 런치** — 시퀀싱 없이 동시 살포 = 모멘텀 희석.
- **H6. 약한 랜딩으로 큰 채널행** — Show HN은 초기 30~90분 승부. 7초 README 없이 침몰.
- **H7. 정체** — 반감기 6주. 갱신 없으면 점유율 이탈.
- **H8. 컬렉션·리포 분산** — 디스커버리·카운트 희석. 한 컬렉션에 모을 것.
- **H9. lattice-fit 외부 주장** — 외부 엔티티(HF·Google·경쟁사·커뮤니티)에 n=6 격자
  파생 규칙 적용 금지(`AGENTS.tape @F f1`·`@D g3`). 그들의 고유 지표만 인용.
